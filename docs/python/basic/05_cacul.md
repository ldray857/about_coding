## 短路与断路效应

### 1. 短路效应（Short-Circuit）

短路指 **逻辑运算符在确定整个表达式结果时，提前停止计算** 的现象。典型逻辑运算符是 `and` 和 `or`：

* **`and`**：如果第一个条件为 `False`，整个表达式一定为 `False`，后面的条件不会再计算。

  ```python
  a = False
  b = 1 / 0   # 如果执行会报错
  print(a and b)  # 输出 False，b 根本没计算
  ```

* **`or`**：如果第一个条件为 `True`，整个表达式一定为 `True`，后面的条件不会再计算。

  ```python
  a = True
  b = 1 / 0
  print(a or b)  # 输出 True，b 根本没计算
  ```

短路效应的好处是 **提高效率**，并且可以避免某些潜在错误（如除零、访问空对象等）。


### 2. 断路效应（Lazy Evaluation）

断路（或惰性求值）指 **表达式或函数的计算被延迟，直到真正需要结果时才执行**。Python 中典型的例子有：

* **生成器（generator）**：

  ```python
  def gen():
      print("计算了")
      yield 1

  g = gen()  # 此时不会打印
  next(g)    # 取值时才打印 "计算了"
  ```

* **`and` / `or` 逻辑运算** 也可以看作一种断路求值，因为只有在必要时才会计算第二个操作数。



## and和or的传输逻辑与返回值

### 1.and运算

**and 的逻辑是从左到右，找到第一个假并返回；如果全部为真，则返回最后一个值。**    

```py
#例1
3 and 5  
```

此例中先算3，是真的  
然后算5也是真的，所以返回5

```py
#例2
3 and (5>3)  
```
此例中先算3，是真的  
然后算(5>3),结果是true也是真的，所以返回ture


```py
#例2
3 and 0 and 5
```
此例中先算3是真的，然后算0，是假的，所以返回这个假值，即0  
然后计算 0 and 5，0是假值，所以返回0


>注意返回的是一个值，可能是数值，也可能是bool


好的，我们按照你讲 `and` 的方式，来讲 **`or` 运算**。`or` 的逻辑是 **从左到右，找到第一个真值并返回；如果全部为假，则返回最后一个值**。


### 2. or`运算

**从左到右依次计算操作数，遇到第一个真值就返回它；如果所有值都为假，返回最后一个假值。**


```python
# 例1
0 or 5
```

先算 `0`，是假值    
再算 `5`，是真值，所以返回 `5`

```python
# 例2
"" or None or "hello"
```

先算空字符串 `""`，是假值  
再算 `None`，是假值  
再算 `"hello"`，是真值，所以返回 `"hello"`

```python
# 例3
0 or False or None
```

* 所有操作数都是假值
* 返回最后一个假值，也就是 `None`


**总结：**

* `and`：找到第一个假值返回，全部为真返回最后一个值
* `or`：找到第一个真值返回，全部为假返回最后一个值
