# JavaScript的作用域和事件

## 1.JavaScript的作用域

### 全局变量

**全局变量 (Global Variables)定义：** 不在任何函数内部或大括号 {} 内部声明的变量。  

**作用域:** 是“全城通用的”。在页面的任何地方、任何脚本、任何函数内部都可以访问它。宿主： 在浏览器中，全局变量会自动成为 window 对象的属性（例如：var a = 1 可以通过 window.a 访问）。    

**周期：** 只要浏览器页面不关闭或刷新，它就一直存在，占据着内存。

**注意：** 如果变量在函数内没有声明（没有使用 var 关键字），该变量为全局变量。

### 局部变量

**局部变量 (Local Variables):** 局部变量可以细分为函数作用域 和 块级作用域. 。

**A. 函数作用域 (Function Scope)** —— var 的领地使用 var 在函数内部定义的变量。  
**作用域：** 只能在该函数内部访问。  
**生命周期：** 函数开始执行时创建，函数执行完毕后，变量通常会被“垃圾回收机制”销毁。     

**B. 块级作用域 (Block Scope)** —— let 和 const 的领地ES6 引入的新特性。在大括号 {}（如 if、for、while 语句块）中定义的变量。  
**作用域：** 仅在这个 {} 内部有效。  
**生命周期：** 进入代码块时诞生，跳出代码块时消失。


## 2.window、document 和 console：浏览器中的三大核心对象

### window：全局宿主对象 (The Boss)
`window` 是浏览器对象模型 (**BOM**, Browser Object Model) 的顶层对象，代表**整个浏览器窗口或标签页**。

*   **核心地位**：它是 JavaScript 在浏览器环境中的**全局对象 (Global Object)**。这意味着你在全局作用域定义的变量或函数，最终都会变成 `window` 的属性。
*   **主要职责**：控制浏览器窗口的行为，管理与网页内容无关的外部环境。
*   **常用功能**：
    *   **交互弹窗**：`window.alert()`, `window.confirm()`。
    *   **视口控制**：`window.innerWidth`, `window.scrollTo()`。
    *   **导航管理**：`window.location` (跳转页面), `window.history` (前进后退)。
    *   **定时器**：`window.setTimeout()`, `window.setInterval()`。

**底层逻辑**：实际上，`document` 和 `console` 都是 `window` 的子属性。即便直接写 `alert()`，浏览器底层执行的也是 `window.alert()`。


### 2. document：文档对象模型 (The Content)
`document` 是文档对象模型 (**DOM**, Document Object Model) 的根节点，代表**当前加载的 HTML 页面内容**。

*   **核心地位**：它是 `window` 的一个属性（`window.document`）。
*   **主要职责**：提供对网页内部元素的访问、修改和结构操作接口。它是你与 HTML 标签交互的唯一“大门”。
*   **常用功能**：
    *   **查找元素**：`document.getElementById()`, `document.querySelector()`。
    *   **动态修改**：`document.title = "新标题"`, `element.style.color = "red"`。
    *   **节点操作**：`document.createElement()`, `element.appendChild()`。



### 3. console：控制台接口 (The Debugger)
`console` 属于浏览器提供的 **Web APIs**，用于访问浏览器的**开发者调试控制台**。

*   **核心地位**：它同样是 `window` 的一个属性（`window.console`）。
*   **主要职责**：面向开发者，用于日志记录、性能监控和错误排查，不会影响普通用户的页面呈现。
*   **常用功能**：
    *   **打印数据**：`console.log()`, `console.dir()` (查看对象结构)。
    *   **状态反馈**：`console.warn()`, `console.error()`。
    *   **性能分析**：`console.time()`, `console.timeEnd()`。


### 📊 核心区别对比表

| 特性 | `window` | `document` | `console` |
| :--- | :--- | :--- | :--- |
| **所属模型** | **BOM** (浏览器对象模型) | **DOM** (文档对象模型) | **Web APIs** (调试接口) |
| **代表含义** | 整个浏览器窗口/容器 | 窗口中承载的具体 HTML 文档 | 浏览器的开发者控制台 |
| **作用范围** | 控制浏览器行为 (尺寸、地址) | 操作网页内容 (文字、图片) | 开发调试 (日志、报错) |
| **层级关系** | **顶层**全局对象 | `window` 的子对象 | `window` 的子对象 |
| **Node.js 环境** | ❌ 无 (Node 使用 `global`) | ❌ 无 | ✅ 有 (Node 实现了自己的 console) |


## 3.明确type

`type` 属性是一个极其重要的“身份标识”，它告诉浏览器如何处理或显示某个元素。它最常出现在 `<input>`、`<button>`、`<ol>` 以及用于引入外部资源的 `<link>` 和 `<script>` 标签中。  
以下是 `type` 属性最常见的几种用途：


### 3.1定义表单控件的功能 (`<input>`)
这是 `type` 属性最强大的地方。通过改变 `type` 的值，同一个 `<input>` 标签可以变成完全不同的控件：

| type 值 | 效果 |
| :--- | :--- |
| `text` | 标准的单行文本框 |
| `password` | 输入内容显示为点或星号（掩码） |
| `checkbox` | 复选框 |
| `radio` | 单选按钮 |
| `file` | 文件上传按钮 |
| `date` | 日期选择器 |
| `submit` | 提交表单的按钮 |



### 3.2设置列表的编号样式 (`<ol>`)
在有序列表 `<ol>` 中，`type` 属性决定了列表项前的编号类型：

*   `type="1"`: 数字 (1, 2, 3) —— **默认值**
*   `type="A"`: 大写字母 (A, B, C)
*   `type="a"`: 小写字母 (a, b, c)
*   `type="I"`: 大写罗马数字 (I, II, III)
*   `type="i"`: 小写罗马数字 (i, ii, iii)


### 3.3声明资源媒体类型 (MIME type)
当引入外部文件时，`type` 告知浏览器文件的格式，以便浏览器决定是直接执行、下载还是忽略。

*   **`<script>`**: 以前常写 `type="text/javascript"`，但在 HTML5 中这已是默认值，通常可以省略。如果要使用 ES 模块，则需写 `type="module"`。
*   **`<link>`**: 在引入 CSS 时，通常指定 `type="text/css"`。
*   **`<style>`**: 定义内部样式表时使用。


### 3.4按钮的行为 (`<button>`)
在表单 `<form>` 内部，`<button>` 的 `type` 属性非常关键：

*   `type="submit"`: 点击后提交表单（**默认行为**）。
*   `type="reset"`: 点击后重置表单内容。
*   `type="button"`: 纯按钮，点击后不触发任何默认行为（通常配合 JavaScript 使用）。





## 4.JavaScript 事件

在 JavaScript 的世界里，**事件 (Events)** 是网页与用户交互的“桥梁”。

### 4.1 什么是 JS 事件

从计算机科学的角度来看，事件是**系统发出的信号**，告知某个特定的行为已经发生。在浏览器中，事件可以是用户执行的操作（如点击、滚动），也可以是浏览器自动触发的状态改变（如页面加载完成、网络断开）。

**事件处理的三要素：**
1.  **事件源 (Event Target)**：谁触发了事件？（例如：一个 `<button>`）
2.  **事件类型 (Event Type)**：触发了什么事？（例如：`click`）
3.  **事件处理程序 (Event Handler)**：触发后要做什么？（例如：执行一个函数）


### 4.2 事件的传播机制 (Event Propagation)

这是 JS 事件最核心的专业概念。当在一个按钮上点击时，事件不仅仅发生在按钮上，它会经历三个阶段：

1.  **捕获阶段 (Capture Phase)**：从 `window` 向下经过各级父元素，直到找到目标元素。
2.  **目标阶段 (Target Phase)**：事件到达目标元素本身。
3.  **冒泡阶段 (Bubble Phase)**：事件从目标元素开始，逐级向上向父元素传播。

> **提示**：大多数情况下我们使用冒泡，因为它符合直觉。你可以通过 `event.stopPropagation()` 来阻止事件继续传播。


### 4.3 JavaScript 事件大分类

JavaScript 的事件有数百种，通常按功能分为以下几大类：

#### ① 鼠标事件 (Mouse Events)
| 事件名 | 触发时机 |
| :--- | :--- |
| `click` | 左键点击 |
| `dblclick` | 双击 |
| `mousedown` / `mouseup` | 鼠标按下 / 抬起 |
| `mousemove` | 鼠标在元素上移动 |
| `mouseenter` / `mouseleave` | 鼠标进入 / 离开元素（不冒泡） |
| `mouseover` / `mouseout` | 鼠标进入 / 离开元素（冒泡） |
| `contextmenu` | 右键点击（弹出菜单前） |

#### ② 键盘事件 (Keyboard Events)
| 事件名 | 触发时机 |
| :--- | :--- |
| `keydown` | 按下键盘按键（最常用） |
| `keyup` | 抬起键盘按键 |
| `keypress` | 字符按键被按下（已废弃，建议用 `keydown`） |

#### ③ 表单事件 (Form Events)
| 事件名 | 触发时机 |
| :--- | :--- |
| `submit` | 表单提交（触发在 `<form>` 上） |
| `input` | 输入内容实时改变时 |
| `change` | 内容改变并失去焦点时 |
| `focus` / `blur` | 获得焦点 / 失去焦点 |
| `reset` | 点击重置按钮时 |

#### ④ 窗口与文档事件 (Resource/Window Events)
| 事件名 | 触发时机 |
| :--- | :--- |
| `load` | 页面所有资源（含图片）加载完成 |
| `DOMContentLoaded` | HTML 结构解析完成（不等待图片，最常用） |
| `resize` | 浏览器窗口大小改变时 |
| `scroll` | 页面或元素滚动时 |
| `error` | 资源加载失败时 |

#### ⑤ 移动端触摸事件 (Touch Events)
*   `touchstart`：手指触摸屏幕。
*   `touchmove`：手指在屏幕上滑动。
*   `touchend`：手指离开屏幕。

#### ⑥ 其他特殊事件
*   **拖拽事件**：`drag`, `dragstart`, `drop`。
*   **剪贴板事件**：`copy`, `cut`, `paste`。
*   **媒体事件**：`play`, `pause`, `ended` (针对视频/音频)。


### 4.4如何监听事件(现代写法)

虽然 HTML 里可以写 `onclick`，但专业开发者通常使用 `addEventListener`，因为它支持同时绑定多个函数，且能控制是在捕获还是冒泡阶段触发。

**示例一**
```javascript
const btn = document.querySelector('button');

// 语法：element.addEventListener(type, listener, useCapture)
btn.addEventListener('click', (event) => {
    console.log("按钮被点击了！");
    console.log("点击的具体坐标：", event.clientX, event.clientY);
}, false); // false 表示在冒泡阶段触发（默认）
```
**示例二**
```js
<!DOCTYPE html>
<html>
<head> 
    <meta charset="utf-8"> 
    <title>更规范的写法</title> 
</head>
<body>

    <button id="timeBtn">现在的时间是?</button>
    <p id="demo"></p>

    <script>
        // 获取按钮元素
        const btn = document.getElementById('timeBtn');
        const display = document.getElementById('demo');

        // 绑定点击事件
        btn.onclick = function() {
            display.innerHTML = Date();
        };
    </script>

</body>
</html>
```


### 4.5核心进阶：事件委托 (Event Delegation)
与其给 100 个列表项 `<li>` 分别绑定事件，不如把事件绑定在它们的父元素 `<ul>` 上。利用**冒泡机制**，父元素可以捕捉到子元素的事件，并通过 `event.target` 判断是哪个子元素被点击了。这能极大地节省内存和提高性能。
