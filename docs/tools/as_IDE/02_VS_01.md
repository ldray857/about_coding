#  VS、VS Code and VS Installer

*引言：* 关于为什么要介绍 VS Code——我相信大部分人都知道 VS Code 或者听过其大名，但常常对于 **VS、VS Code、VS Installer** 三者的定义模糊。希望通过这个文档，你能对三者有更清晰的理解 💡。

---

## 1️⃣ 背景公司介绍 🏢

**微软（Microsoft）**：在早年间，微软的工具（如 Visual Studio）主要服务于 Windows 平台的开发者。但自从 **萨提亚·纳德拉（Satya Nadella）** 接任 CEO 后，微软的策略发生巨大变化，口号变成了“微软爱开源” 。

VS Code 是这一转型的代表作。微软意识到开发者不再只用 Windows，很多人在 Mac 或 Linux 上写代码 。于是他们基于开源技术（Electron）打造了 VS Code，并将其代码在 GitHub 上开源 。

---

## 2️⃣ 产品基本介绍

### 2.1 Visual Studio Code (VS Code) 📝

**定位**：开源、跨平台的 **现代化代码编辑器**。  

**核心特点**：
* ⚡ **轻量与极速**：启动极快，内存占用低  
* 🔌 **插件库齐全**：安装扩展即可完成各种轻量开发  
* 🌍 **跨平台**：Windows、macOS 和 Linux 上体验一致  

💡 对于普通 coding，常用 VS Code。关于我的使用经验，可以查看：[VScode入门](VScode/)

<br/>

### 2.2 Visual Studio (VS) 🏗️

**定位**：被誉为“宇宙第一 IDE”的 **全功能集成开发环境**  

**核心特点**：
* 🏋️ **重型 IDE**：内置强大编译器、调试器、性能分析器和图形化界面设计器  
* 🔗 **深度集成**：工业级对 .NET 和 C++ 支持，适合管理大型项目  
* 🪟 **Windows 原生**：功能最强始终在 Windows  
* 🎯 **适用场景**：大型 Windows 桌面程序、专业游戏开发（如 Unreal Engine）、复杂 C++ 算法研究  

<br/>

### 2.3 Visual Studio Installer ⚙️

**定位**：用于安装、管理和修改 Visual Studio 的官方工具  

**主要功能**：
* 📦 **按需取用**：通过勾选“工作负载”安装需要的组件，节省磁盘空间  
* 🔄 **一键更新**：管理 VS 版本升级和修复  

!!! 
    它服务于 Visual Studio；而VS Code 通常独立安装，不通过 Installer 管理。

---

### 🔍 三者对比

| 特性 | VS Code 📝 | Visual Studio 🏗️ | VS Installer ⚙️ |
| --- | --- | --- | --- |
| **工具类型** | 代码编辑器 | 集成开发环境 (IDE) | 安装/管理工具 |
| **资源占用** | ⚡ 较低 | 🏋️ 较高 | 🪄 极低 |
| **上手难度** | 😊 简单，即装即用 | ⚙️ 较复杂，配置多 | 😊 简单 |
| **核心优势** | 🔌 灵活、生态丰富 | 🏆 功能完备、调试强大 | 📦 模块化安装、易于维护 |
